Heaps & Priority Queues

Min/Max heap structures, dynamic tracking of extreme elements, and continuous K-way merges.

💡 Core Mechanics

  • A Heap is a Complete Binary Tree mapped internally to a flat array layout to maintain fast cache locality.
  • Time Complexities: Accessing the Min/Max element is O(1). Inserting an element or extracting the root takes O(log N). Transforming an unsorted array into a heap (Heapify) runs in O(N).
  • Visual Cue: Reach for a Heap whenever a problem asks you to repeatedly track the "top K", "K-th largest/smallest", or continuously fetch the minimum/maximum element from a dynamic stream of incoming data.

1. Unified Heap Implementation (Min & Max)

Medium

Implement a flexible Heap class in JavaScript that supports both Min-Heap and Max-Heap variants using a constructor flag. The class manages structural invariants dynamically through array index arithmetic.

The "Aha!" Moment: Instead of duplicating the entire codebase for Min and Max layouts, wrap the comparison operations in a helper method or check a configuration variable. The underlying swap mechanics, parent/child calculations, and array restructuring remain completely identical.

// Visual Trace Loop / Whiteboard Diagram

class Heap {
    constructor(type = 'min') {
        this.heap = [];
        this.type = type; // 'min' or 'max'
    }

    getLeftChildIndex(i) {
        return 2 * i + 1;
    }

    getRightChildIndex(i) {
        return 2 * i + 2;
    }

    getParentIndex(i) {
        return Math.floor((i - 1) / 2);
    }

    // Helper to abstract the comparison logic based on the heap type
    compare(a, b) {
        // Min-Heap: parent should be smaller than child
        // Max-Heap: parent should be larger than child
        return this.type === 'min' ? a < b : a > b;
    }

    heapifyUp(idx) {
        let i = idx;

        while (i > 0) {
            const parentIdx = this.getParentIndex(i);

            // Compare child with parent based on Min/Max rules
            if (this.compare(this.heap[i], this.heap[parentIdx])) {
                [this.heap[i], this.heap[parentIdx]] = 
                    [this.heap[parentIdx], this.heap[i]];
                i = parentIdx;
            } else {
                break;
            }
        }
    }

    heapifyDown() {
        let i = 0;

        while (true) {
            const leftIdx = this.getLeftChildIndex(i);
            const rightIdx = this.getRightChildIndex(i);
            let targetIdx = i; // Tracks smallest (MinHeap) or largest (MaxHeap)

            // Check if left child breaks the invariant
            if (
                leftIdx < this.heap.length &&
                this.compare(this.heap[leftIdx], this.heap[targetIdx])
            ) {
                targetIdx = leftIdx;
            }

            // Check if right child breaks the invariant
            if (
                rightIdx < this.heap.length &&
                this.compare(this.heap[rightIdx], this.heap[targetIdx])
            ) {
                targetIdx = rightIdx;
            }

            if (targetIdx !== i) {
                [this.heap[i], this.heap[targetIdx]] = 
                    [this.heap[targetIdx], this.heap[i]];
                i = targetIdx;
            } else {
                break;
            }
        }
    }

    insertNode(val) {
        this.heap.push(val);
        this.heapifyUp(this.heap.length - 1);
        return val;
    }

    extractNode() {
        if (this.heap.length === 0) return null;
        if (this.heap.length === 1) return this.heap.pop();

        const rootVal = this.heap[0]; // Extracted minimum or maximum element
        this.heap[0] = this.heap.pop();
        this.heapifyDown();

        return rootVal;
    }

    peek() {
        return this.heap[0] ?? null;
    }

    size() {
        return this.heap.length;
    }
}

2. Kth Largest Element in an Array (LeetCode #215)

Medium

Find the k-th largest element in an unsorted array. Note that it is the k-th largest element in the sorted order, not the k-th distinct element.

The "Aha!" Moment: To find the **largest** elements, maintain a **Min-Heap** limited to size k. By constantly eviction-filtering the smallest tracked value whenever the heap size grows larger than k, the heap is guaranteed to hold the k absolute largest elements seen so far. The minimum value remaining at the root will be your exact target answer. 
var findKthLargest = function(nums, k) {
    // Relying on native LeetCode MinPriorityQueue collection wrapper
    const pq = new MinPriorityQueue();

    for (const num of nums) {
        pq.enqueue(num);
        
        // If heap size passes K, kick out the absolute smallest element.
        // This ensures the smallest numbers are purged, leaving the top K largest.
        if (pq.size() > k) {
            pq.dequeue();
        }
    }

    // The root of our Min-Heap is now the smallest of the K largest elements
    return pq.front().element;
};

3. Kth Largest Element in a Stream (LeetCode #703)

Easy

Design a class to find the k-th largest element in a stream of numbers. The class should process initial numbers and efficiently handle new numbers added dynamically over time, always returning the current k-th largest element.

The "Aha!" Moment: A data stream means numbers arrive indefinitely, making sorting techniques completely unusable. By maintaining a persistent instance-level Min-Heap restricted to size k, the initialization runs in $O(N \log K)$ time, and each subsequent data insertion drops to a blazing-fast **$O(\log K)$ operational speed**. 
/**
 * @param {number} k
 * @param {number[]} nums
 */
var KthLargest = function(k, nums) {
    this.pq = new MinPriorityQueue();
    this.k = k;

    // Stream incoming array data points through our internal pipeline
    for (const num of nums) {
        this.add(num);
    }
};

/** * @param {number} val
 * @return {number}
 */
KthLargest.prototype.add = function(val) {
    this.pq.enqueue(val);
    
    // Evict the lowest value to keep only the top K largest values in memory
    if (this.pq.size() > this.k) {
        this.pq.dequeue();
    }

    // Return the root of the Min-Heap (the small boundary of our top-K shelf)
    return this.pq.front().element;
};

/**
 * Usage Reference:
 * const kthLargest = new KthLargest(3, [4, 5, 8, 2]);
 * kthLargest.add(3);   // returns 4
 * kthLargest.add(5);   // returns 5
 */

4. Last Stone Weight (LeetCode #1046)

Easy

You are given an array of integers stones where stones[i] is the weight of the i-th stone. We smash the two heaviest stones together. If they are of equal weight, both are destroyed. If not, a new stone with a weight equal to the difference is smashed back into the collection. Repeat until at most one stone remains.

The "Aha!" Moment: This is a dynamic simulation problem where the collection structure updates after every step. Since you constantly need to pull the two largest weights, use a Max-Heap. Extracting the top two elements takes $O(\log N)$ time, and smashing them back takes $O(\log N)$ time, keeping the overall simulation loop highly optimized at $O(N \log N)$ total speed. 
var lastStoneWeight = function(stones) {
    // Relying on native LeetCode MaxPriorityQueue collection wrapper
    const pq = new MaxPriorityQueue();

    for (const stone of stones) {
        pq.enqueue(stone);
    }

    // Smash stones together until 0 or 1 stone remains
    while (pq.size() > 1) {
        // MaxPriorityQueue returns nodes with .element and .priority properties
        const stoneOne = pq.dequeue().element;
        const stoneTwo = pq.dequeue().element;
        const absDiff = Math.abs(stoneOne - stoneTwo);

        // If stones aren't equal, a smaller fragment remains; add it back to the heap
        if (absDiff !== 0) {
            pq.enqueue(absDiff);
        }
    }

    // If a final stone remains, return its raw weight value, otherwise return 0
    return pq.size() ? pq.front().element : 0;
};

5. Top K Frequent Elements (LeetCode #347)

Medium

Given an integer array nums and an integer k, return the k most frequent elements. You may return the answer in any order.

The "Aha!" Moment: This problem blends two core patterns. First, scan the array into a Hash Map to compute element frequencies in $O(N)$ time. Next, stream those hash map frequencies through a Min-Heap capped at size k. By configuring the priority queue to compare values based on their frequency property (x => x.freq), the heap automatically filters out the least frequent keys, preserving only the k most frequent elements at optimal efficiency. 
var topKFrequent = function(nums, k) {
    const map = {};
    
    // Step 1: Count element frequencies using a hash map
    for (const num of nums) {
        map[num] = (map[num] || 0) + 1;
    }

    // Step 2: Use LeetCode's Priority Queue, passing a callback to prioritize by frequency count
    const pq = new MinPriorityQueue({ priority: x => x.freq });
    
    for (const key in map) {
        pq.enqueue({ value: key, freq: map[key] });
        
        // If the heap grows larger than K, pop the element with the absolute lowest frequency
        if (pq.size() > k) {
            pq.dequeue();
        }
    }

    // Step 3: Extract leftover node elements, conversion to native Numbers is mandatory
    return pq.toArray().map(v => Number(v.element.value));
};

6. Kth Smallest Element in a Sorted Matrix (LeetCode #378)

Medium

Given an n x n matrix where each of the rows and columns is sorted in ascending order, return the k-th smallest element in the matrix. Note that it is the k-th smallest element in the sorted order, not the k-th distinct element.

The "Aha!" Moment: Instead of flattening the matrix ($O(N^2)$) or running a full sorting pass, treat this as a K-Way Merge. Since every row is already sorted, the absolute smallest element must reside in the very first column. Initialize a Min-Heap with the first element of every row. Pop the minimum element exactly k - 1 times. Each time you pop an element, insert its immediate right neighbor from the same row into the heap. The k-th pop guarantees your target value at an optimized speed of $O(K \log N)$. 
var kthSmallest = function(matrix, k) {
    const n = matrix.length;
    // Configure LeetCode priority queue constructor using object priorities
    const heap = new MinPriorityQueue({ priority: x => x.val });
    
    // Step 1: Initialize heap with the start boundaries of all rows
    for (let i = 0; i < n; i++) {
        heap.enqueue({ val: matrix[i][0], row: i, col: 0 });
    }

    // Step 2: Pop the minimum root element K-1 times
    for (let count = 0; count < k - 1; count++) {
        // Extract matrix location tracking markers from internal heap node payload
        const { val, row, col } = heap.dequeue().element;
        
        // If an element exists to the right of the popped item, insert it
        if (col + 1 < n) {
            heap.enqueue({ val: matrix[row][col + 1], row, col: col + 1 });
        }
    }

    // Step 3: The root of the Min-Heap is now precisely the K-th smallest value
    return heap.front().element.val;
};
