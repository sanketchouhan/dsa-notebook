Sorting Algorithms

Methods for arranging linear array data into ascending or descending orders efficiently.

💡 Core Metrics & Trade-offs

  • Stability: A sorting algorithm is stable if it preserves the relative order of duplicate elements.
  • In-Place: Algorithms that sort data using O(1) auxiliary space without cloning the input collection.
  • Comparison Limit: Comparison-based sorting algorithms have a theoretical lower boundary execution speed limit of O(N log N).

2. Bubble Sort (Sink Sort)

Easy

Implement Bubble Sort in JavaScript. Iterate through the array repeatedly, comparing adjacent elements and swapping them if they are in the wrong order. The largest elements "bubble" up to their correct positions at the end of the array with each pass.

The "Aha!" Moment: Standard Bubble Sort runs in $O(N^2)$ even if the array is already sorted. Optimize it to **$O(N)$ best-case time** by tracking if a swap occurred during a pass. If no elements swap positions during an entire pass, the array is completely sorted, and you can break out early. 
function bubbleSort(arr) {
    const n = arr.length;
    
    for (let i = 0; i < n; i++) {
        let swapped = false;
        
        // Last i elements are already in place
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap adjacent elements
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
                swapped = true;
            }
        }
        
        // Optimization: If no elements were swapped, array is sorted
        if (!swapped) break;
    }
    return arr;
}

3. Selection Sort (Scan & Swap)

Easy

Implement Selection Sort in JavaScript. Divide the array conceptually into a sorted and an unsorted boundary. Repeatedly scan the unsorted portion to find the absolute minimum element, then swap it into its finalized position at the boundary.

The "Aha!" Moment: Unlike other quadratic sorting algorithms, Selection Sort minimizes writing data. It performs at most **$O(N)$ total memory swaps**, making it highly efficient in environments where modifying memory pages or flash blocks is computationally expensive. 
function selectionSort(arr) {
    const n = arr.length;

    for (let i = 0; i < n - 1; i++) {
        let minIndex = i;

        // Scan the unsorted array block to find the smallest element
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }

        // Swap the minimum found element with the current boundary index
        if (minIndex !== i) {
            [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
        }
    }
    return arr;
}

4. Insertion Sort (Card Deck Shift)

Easy

Implement Insertion Sort in JavaScript. Build a sorted sub-segment at the front of the array one item at a time. Pick the next unsorted element and shift elements within the sorted section to the right until the correct slot is cleared to insert the item.

The "Aha!" Moment: Think of it like organizing a hand of playing cards. Insertion Sort is an **online algorithm**, meaning it can process an active incoming stream of numbers one by one and keep its state completely sorted at any given moment. 
function insertionSort(arr) {
    const n = arr.length;

    for (let i = 1; i < n; i++) {
        let current = arr[i];
        let j = i - 1;

        // Shift elements of the sorted partition rightward 
        // until a value smaller than the current element is reached
        while (j >= 0 && arr[j] > current) {
            arr[j + 1] = arr[j];
            j--;
        }
        
        // Place the extracted value into its cleared slot
        arr[j + 1] = current;
    }
    return arr;
}

5. Heap Sort (In-Place Tree Selection)

Medium

Implement Heap Sort in JavaScript. Transform the unsorted input array into a Max-Heap structure, then repeatedly extract the maximum element from the root of the heap and place it at the shrinking end boundary of the array.

The "Aha!" Moment: Heap Sort is essentially an optimized Selection Sort. Instead of scanning the entire unsorted block linearly ($O(N)$) to find the largest value, it builds a Max-Heap to lookup and retrieve the maximum value in **$O(1)$** time, then restores the structure in **$O(\log N)$** time. This guarantees a strict **$O(N \log N)$ worst-case runtime** using $O(1)$ auxiliary space. 
function heapSort(arr) {
    const n = arr.length;

    // Step 1: Build a Max-Heap out of the input array
    // Start from the last non-leaf node and heapify down each element
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    // Step 2: Extract elements from the heap one by one from the back
    for (let i = n - 1; i > 0; i--) {
        // Move current root (absolute maximum element) to the end boundary
        [arr[0], arr[i]] = [arr[i], arr[0]];

        // Call max heapify on the reduced heap area to restore invariant
        heapify(arr, i, 0);
    }
    return arr;
}

// In-place Max-Heapify sub-routine matching array size limitations
function heapify(arr, size, i) {
    let largest = i;
    const left = 2 * i + 1;
    const right = 2 * i + 2;

    if (left < size && arr[left] > arr[largest]) {
        largest = left;
    }

    if (right < size && arr[right] > arr[largest]) {
        largest = right;
    }

    if (largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]]; // Swap breaking element down
        
        // Recursively heapify the affected sub-tree element structure
        heapify(arr, size, largest);
    }
}
